# Gitnexus MCP 代码检索工具安装与使用指南

Gitnexus MCP Server 提供跨项目代码检索能力，是代码排查的首选工具。

## 前置检查

调用前确认 MCP 工具列表中是否有 `mcp__gitnexus` 相关工具。

## 安装配置

### CodeBuddy MCP 配置

在 `~/.codebuddy/.mcp.json` 中添加：

```json
{
  "mcpServers": {
    "gitnexus": {
      "url": "http://10.3.8.8:4748/api/mcp"
    }
  }
}
```

完成后重启 CodeBuddy 或重连 MCP server 即可。

## 工具优先级

在需要代码分析时，按以下优先级选择工具：

1. **优先** — `mcp__gitnexus` MCP 工具：跨项目代码检索，速度快、覆盖面广
2. **降级** — 确认不使用 MCP 工具，或 gitnexus 中没有对应项目时，使用本地仓库 `grep`/`glob` 搜索

## 使用场景

### 需求分析
- 搜索需求涉及的服务/模块入口（Controller、API 路由等）
- 搜索相关的 Service 实现、DAO 层、配置项
- 搜索同类功能的已有实现作为开发参考

### 技术方案设计
- 搜索涉及的 Service 实现、DAO 层、ORM 映射
- 搜索相关模块的分层结构和依赖注入方式
- 搜索同类功能的技术方案实现
- 搜索现有中间件、过滤器、拦截器的使用方式

### Spec 编写
- 搜索现有 API 的 Request/Response 结构体定义（DTO、VO 类）
- 搜索现有 Controller 的路由定义和参数校验方式
- 搜索同类功能的接口设计模式（命名规范、分页方式、错误码格式）

### 开发实现
- 搜索涉及的服务/模块入口、关键函数名、接口路径
- 搜索相关配置项、常量定义
- 搜索同类功能的已有实现作为参考

### 联调排错
- 搜索接口定义、出入参结构、序列化逻辑
- 搜索中间件、拦截器、过滤器对请求/响应的处理
- 搜索跨模块调用链路的传递过程

### Bug 修复
- 搜索 Bug 涉及的接口路径、异常类名、错误信息关键词
- 搜索相关模块的入口函数、配置项
- 追踪调用链路，找出异常发生点
- 检查：异常处理、超时配置、资源释放、并发控制、边界条件

## 注意事项

- 优先使用 `mcp__gitnexus` 而非直接 grep/glob，前者更适合跨项目检索
- gitnexus 中可能没有所有项目，确认不存在时再降级
