## 当前任务

向提审人说明当前页面不支持审核或审核已完成，无法继续执行审核流程。

## 上下文

- 提审人：`{{requester}}`
- 审核页面：`{{sql_review_url}}`
- 页面状态：{{steps.open_check_page.outputs.page_status}}

## 执行步骤

1. 根据页面状态分析无法审核的原因：
   - **页面不包含审核按钮**：该页面不是一个有效的 SQL 审核页面
   - **审核已提交/完成**：页面状态显示"已通过"、"已拒绝"、"已关闭"等已完成状态
   - **权限不足**：当前登录用户无审核权限
   - **页面错误**：页面无法正常加载（404、500 等）

2. 整理清楚的通知消息：

   ```
   ## SQL 审核无法进行

   审核页面：{{sql_review_url}}
   检查结果：[根据 page_status 说明具体情况]

   原因：[根据 page_status 分析的具体原因]

   建议：[给出后续操作建议，如：联系管理员开通权限、检查页面地址是否正确、确认审核是否已由他人完成]
   ```

3. 将消息通知给提审人 `{{requester}}`。

## 产出物

调用 `flow_advance` 时提供：

- **notified**：`true`（确认已通知）
- **reason**：字符串（至少 10 个字符），无法审核的详细原因描述
